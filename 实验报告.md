# 简易天气预报 App 实验报告

---

## 一、实验目的

**老师：** 这次实验你们要做一个简易天气预报应用，主要目标是什么？

**学生：** 老师，这次实验主要有以下几个目的：

1. **掌握 Android 项目的基本结构**：从零搭建一个完整的 Android 工程，理解 `build.gradle`、`AndroidManifest.xml`、`settings.gradle` 等配置文件的作用，熟悉 Android 项目的目录组织方式。

2. **熟练使用常用 UI 组件**：运用 `MaterialCardView`、`RecyclerView`、`LinearLayout`、`TextView`、`ImageView` 等组件，搭建美观的天气展示界面，掌握布局嵌套和属性配置。

3. **实现数据与视图的分离**：通过 `WeatherData` 和 `DailyForecast` 数据模型类，将天气数据与界面展示解耦，理解 MVC 架构思想。

4. **掌握 RecyclerView 列表开发**：编写 `ForecastAdapter` 适配器，实现未来 7 天天气预报列表的动态渲染，理解 `ViewHolder` 模式和数据绑定流程。

5. **实现定时刷新机制**：使用 `Handler` 和 `Runnable` 实现每秒刷新时钟，掌握 Android 中的消息机制和定时任务处理。

6. **综合运用条件逻辑**：根据不同天气类型（晴、雨、雪、雾等）智能生成"今日宜/不宜"生活建议，锻炼业务逻辑设计能力。

7. **适配旧版开发环境**：针对 AGP 7.3.1 + Gradle 7.4 的旧版 Android Studio 进行配置，使用国内镜像源解决依赖下载问题，积累实际工程踩坑经验。

## 三、实验步骤

**老师：** 好的，那你详细讲一下整个开发过程吧。

**学生：** 好的老师，整个开发过程分为以下几步：

### 步骤一：创建项目工程结构

首先搭建 Android 项目的基础骨架。手动创建了以下配置文件：

- **`build.gradle`（项目级）**：指定 AGP 7.3.1 插件版本，配置阿里云 Maven 镜像仓库地址。
- **`settings.gradle`**：配置 `pluginManagement` 和 `dependencyResolutionManagement` 的镜像源，声明项目包含 `:app` 模块。
- **`gradle.properties`**：设置 JVM 内存参数 `-Xmx2048m`，开启 AndroidX 和 Jetifier。
- **`gradle/wrapper/gradle-wrapper.properties`**：指定使用腾讯云镜像下载 Gradle 7.4 发行包。
- **`local.properties`**：配置本机 Android SDK 路径。

### 步骤二：配置 app 模块

创建 `app/build.gradle`，配置命名空间 `com.example.simpleweatherforecast`，设置 `compileSdk 33 / minSdk 21 / targetSdk 33`，声明所需的 Material Design、RecyclerView 等依赖库。

编写 `AndroidManifest.xml`，声明 `INTERNET` 和 `ACCESS_NETWORK_STATE` 权限（为后续接入真实 API 预留），注册 `MainActivity` 作为启动 Activity。

### 步骤三：设计数据模型

创建两个数据模型类：

- **`DailyForecast.java`**：单日预报数据模型，包含日期、星期、天气描述、最高/最低温度、湿度、风力等级、风向、图标资源 ID、宜/不宜建议列表等字段，配套 getter/setter 方法。
- **`WeatherData.java`**：整体天气数据容器，包含城市名、更新时间、今日天气详情和未来多日预报列表。

### 步骤四：编写 Mock 天气数据生成器

创建 `MockWeatherData.java` 工具类，这是项目的核心数据逻辑：

1. **季节性温度**：根据当前月份判断季节（春3-5月、夏6-8月、秋9-11月、冬12-2月），返回对应的温度范围。
2. **天气类型随机**：从 8 种天气类型（晴、多云、阴、小雨、中雨、雷阵雨、小雪、雾）中随机选取。
3. **湿度计算**：不同天气对应不同湿度区间，如晴天 30-50%，雨天 80-95%。
4. **宜/不宜建议生成**：核心方法 `generateSuggestions()`，根据天气类型匹配预设建议，例如：
   - 晴天 → 宜户外运动、晒太阳、晾晒衣物；不宜长时间暴晒
   - 雷阵雨 → 宜关好门窗、拔掉电器插头；不宜户外活动、树下避雨
   - 雾天 → 宜室内活动、佩戴口罩；不宜驾车出行
5. **额外逻辑**：根据湿度高低补充建议，如湿度 >80% 时不宜存放易受潮物品，湿度 <40% 时宜使用加湿器。

### 步骤五：编写 RecyclerView 适配器

创建 `ForecastAdapter.java`：

1. 继承 `RecyclerView.Adapter`，泛型为自定义 `ViewHolder`。
2. `ViewHolder` 中持有日期、星期、天气图标、天气描述、温度、风力、湿度等控件的引用。
3. `onBindViewHolder()` 中将 `DailyForecast` 数据绑定到对应控件上。
4. 提供 `updateData()` 方法支持数据刷新。

### 步骤六：编写 MainActivity 主界面

创建 `MainActivity.java`：

1. **初始化控件**：`initViews()` 方法通过 `findViewById` 获取所有界面控件的引用。
2. **加载数据**：`loadWeatherData()` 调用 `MockWeatherData.generateWeatherData("北京")` 获取模拟天气数据，然后更新今日天气卡片。
3. **动态生成宜/不宜**：`updateSuggestions()` 方法遍历建议列表，动态 `inflate` 子布局 `item_suggestion.xml` 并添加到对应的 `LinearLayout` 容器中。
4. **实时时钟**：使用 `Handler` + `Runnable` 机制，每秒调用 `updateDateTime()` 方法刷新时间和日期/星期显示，在 `onDestroy()` 中移除回调防止内存泄漏。
5. **初始化列表**：设置 `RecyclerView` 的 `LinearLayoutManager` 和 `ForecastAdapter`。

### 步骤七：设计界面布局

创建三个布局文件：

- **`activity_main.xml`**：主界面布局，采用 `ScrollView` + `LinearLayout` 垂直排列。顶部显示城市名、实时时间和日期星期；中间是半透明天气大卡片（大温度 + 天气描述 + 湿度/风力标签）；下方是左右分栏的宜/不宜区域（绿色底/红色底卡片）；底部是"近期天气预报"标题和 `RecyclerView` 列表。整体背景使用蓝色渐变 `bg_sky_gradient`。
- **`item_forecast.xml`**：单条预报列表项，`MaterialCardView` 包裹横向布局，左侧日期星期、中间天气图标+描述、右侧温度+风力+湿度。
- **`item_suggestion.xml`**：单个建议条目，一个小圆点 + 文字。

### 步骤八：创建资源文件

1. **颜色 `colors.xml`**：定义天空蓝、卡片背景色、宜/不宜文字色等。
2. **主题 `themes.xml`**：继承 `Theme.MaterialComponents.Light.NoActionBar`，设置状态栏颜色为天空蓝。
3. **字符串 `strings.xml`**：应用名称"简易天气预报"。
4. **Drawable 资源**：
   - 背景渐变 `bg_sky_gradient.xml`（蓝色渐变）
   - 卡片背景 `bg_card_suitable.xml`（浅绿）、`bg_card_unsuitable.xml`（浅红）
   - 标签背景 `bg_tag_translucent.xml`（半透明圆角）
   - 小圆点 `bg_dot.xml`
5. **矢量图标**：
   - 宜/不宜图标：`ic_check.xml`（绿色对勾）、`ic_cross.xml`（红色叉号）
   - 8 个天气图标：`ic_weather_sunny.xml`、`ic_weather_cloudy.xml`、`ic_weather_overcast.xml`、`ic_weather_light_rain.xml`、`ic_weather_rain.xml`、`ic_weather_thunder.xml`、`ic_weather_snow.xml`、`ic_weather_fog.xml`
6. **应用图标**：自适应图标 `ic_launcher.xml`（含前景太阳+云朵图案和蓝色背景），兼容 API 26+ 和旧版设备。

### 步骤九：编译运行与调试

1. 在 Android Studio 中打开项目，等待 Gradle Sync 完成。
2. 连接 Android 模拟器或真机，点击运行按钮。
3. 验证各功能：温度显示、实时时钟每秒刷新、宜/不宜建议随天气变化、7 天预报列表正常滚动、天气图标正确匹配。
4. 修复编译错误和 UI 问题（如 `xmlns:app` 命名空间声明、`ScrollView` 内 `RecyclerView` 嵌套滚动冲突等）。

**老师：** 很完整。那你觉得这个项目里最有挑战性的部分是什么？

**学生：** 主要有两个难点：

第一是 **RecyclerView 在 ScrollView 中的嵌套问题**。因为整个页面需要滚动，而预报列表也是可滚动的 `RecyclerView`，直接嵌套会导致列表高度不正确。解决方案是设置 `android:nestedScrollingEnabled="false"` 和 `android:overScrollMode="never"`，让 `RecyclerView` 不自己滚动，完全展开在 `ScrollView` 中。

第二是 **宜/不宜建议的动态生成**。不是简单地写死文字，而是要根据天气类型、温度、湿度等多个因素综合判断，用 `switch-case` 分天气类型匹配预设建议列表，再根据湿度额外补充条目，保证建议合理且贴近实际生活。
